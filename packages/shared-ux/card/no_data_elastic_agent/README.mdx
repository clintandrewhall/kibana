## Description

A wrapper around `NoDataCard` that prompts a person to add an Elastic Agent.  All of the `NoDataCard` props are available, with `category` and `canAccessFleet` included.

The `NoDataElasticAgentCard` connected component uses:

- `navLinks.integrations` from `coreStart.application.capabilities` to determine if the user has access to the Integrations page.
- `addBasePath` from `coreStart` to navigate to the Integrations page.

## API
| Export | Description |
|---|---|
| `NoDataElasticAgentCardProvider` | Provides contextual services to `NoDataElasticAgentCard`. |
| `NoDataElasticAgentCardKibanaProvider` | Maps Kibana dependencies to provide contextual services to `NoDataElasticAgentCard`. |
| `NoDataElasticAgentCard` | Uses a `Provider` to access contextual services to populate props on the `NoDataElasticAgentCardComponent`. |
| `NoDataElasticAgentCardComponent` | The pure component, a pre-configured **NoDataCard**. |
